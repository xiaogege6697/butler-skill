# Dispatch Templates — 常用派发模板

## 模板1：并行侦察（最常用）

适用：多方向信息搜集

```
目标：调研{主题}
输入：关键词「{关键词}」
约束：
  - 找{N}条相关结果
  - 每条包含：标题、来源URL、一句话摘要
  - 输出JSON数组格式
验收：
  - 至少{N}条结果
  - 每条URL可访问
  - 摘要不超过50字
超时：{N}分钟，超了返回已找到的部分
```

派发方式：N个侦察兵并行，每个负责一个方向。管家最后汇总去重。

## 模板2：串行流水线

适用：有明确步骤的生成任务

```
Step 1: 收集原料 → raw-data.json
Step 2: 加工整理 ← raw-data.json → draft.md
Step 3: 质检包装 ← draft.md → final.md（验收：格式正确、内容完整）
```

## 模板3：带校验的单任务

适用：需要质量保证的生成任务

```
目标：生成{产出物}
输入：{输入描述}
约束：{格式/内容要求}
验收标准：条件1、条件2、条件3
如果不符合，说明哪里不符合，附上原始输出退回重做。
```

## 模板4：降级链

适用：核心流程的容错

```
主方案：{理想路径}
失败 → 降级A：{简化版本}
失败 → 降级B：{最小可用版本}
失败 → 报告用户，附分析
```
